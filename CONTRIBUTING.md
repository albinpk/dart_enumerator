# Contributing

Thanks for your interest in contributing to **enumerator**.

This project aims to stay **small, focused, and predictable**, so contributions should align with that goal.

---

## Ways to Contribute

- Report bugs or unexpected behavior
- Suggest improvements or new features
- Improve documentation or examples
- Submit bug fixes or focused enhancements

---

## Before Submitting a PR

Please ensure that:

- The change has a clear use case
- The scope is limited and well-defined
- Existing behavior is not broken
- Public APIs remain backward compatible

For larger changes or new features, itâ€™s recommended to open an issue first.

---

## Code Guidelines

- Follow standard Dart formatting and style
- Keep generated code deterministic and readable
- Avoid unnecessary abstractions
- Prefer compile-time solutions over runtime logic

---

## Testing

- Add or update tests where applicable
- Ensure `build_runner` completes without errors
- Verify generated output for both default and configured cases

---

## Commit Messages

- Use clear, descriptive commit messages
- Reference issues when relevant
